<objective>
Deliver the "release → drift-detected" half of Phase 3's core loop as one vertical slice: with both workaround versions live, the dataset list surfaces the superseded workaround as 🔴 Drifting with a human-readable, Cognee-generated reason, and `/search` stops treating that drifting dataset as the primary root-cause answer.

Purpose: This is the scoring differentiator's first visible capability (DRIFT-01/02/03) and the load-bearing D-01 behavior — the root-cause selection must exclude drifting datasets, resolving the FEEDBACK-02-class "highest version always wins regardless of user action" finding Phase 2's verifier documented.

Output: A restored demo corpus (both `workarounds_v1_8` and `workarounds_v1_9` live), a new `backend/drift.py` shared classifier + live reason generator, an extended `_pick_primary_result` + `GET /datasets`, and the frontend badge/reason rendering wired into the reserved slots Phase 2 left for exactly this.

MVP framing: this plan is a single vertical slice (drift detection + its visible rendering). Task 1 is the enabling corpus restore; task 2 makes the API return drift facts and change search behavior; task 3 renders those facts. After task 3 a real user can search a bug and see the old workaround flagged 🔴 with a reason it never showed before.
</objective>
