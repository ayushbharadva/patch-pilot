<objective>
Deliver the "forget → re-search proves it" half of Phase 3's core loop as one vertical slice: the engineer clicks Forget on a 🔴 drifting dataset, confirms once inline, and the workaround is surgically removed via `cognee.forget(dataset=...)` — the row vanishes from the list and the last search auto-re-runs, making the before/after change visible.

Purpose: Closes FORGET-01 (surgical, guarded forget) and FORGET-02 (the visible before/after proof), completing the search → drift → forget → re-search loop that is PatchPilot's whole reason to exist.

Output: A new `backend/forget.py` mirroring `feedback.py`'s validate-before-lifecycle-verb pattern (plus a durable-dataset guard Cognee itself does not provide), registered in `main.py`; a `forgetDataset()` client wrapper; and a two-step-confirm `ForgetButton` in the dataset list wired to auto-re-search via the page's existing `handleReSearch`.

MVP framing: one vertical slice (surgical forget + proof). Task 1 makes the destructive action safe and real end-to-end (validated endpoint → Cognee); task 2 gives the user the guarded button and the automatic re-search that renders the proof. After task 2 a real user can do something impossible before: retire a stale workaround in one guarded click and immediately see memory change.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering / Elevation of Privilege | backend/forget.py — forged dataset name reaching `cognee.forget(dataset=...)` (e.g. `"incidents"` or a crafted string) | high | mitigate | Validate-before-lifecycle-verb (Pattern 5): `workarounds_v{N}` regex allowlist + live-existence check via `list_datasets()` + explicit durable-dataset denylist (`incidents`/`healthcheck`/`canary`). `cognee.forget()` is never reached for a name that fails this guard, so `incidents` (INGEST-03/D-01 durability) can never be forgotten through this route. |
| T-03-02 | Information Disclosure | backend/forget.py error path (Pitfall 2 raw `AttributeError`) | medium | mitigate | Regex pre-validation prevents the `NoneType.id` `AttributeError` entirely; a broad `except Exception` + `logger.exception` + fixed short message (`Could not forget dataset. Please try again.`) is defense-in-depth so no raw exception text ever reaches the client (D-24). |
| T-03-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages this phase (RESEARCH § Package Legitimacy Audit: "Not applicable"); `Trash2` is an already-installed `lucide-react` icon. No install task exists. |
