## Phase Goal

**As an** engineer, **I want to** ingest incident history, search a bug, and get an evidence-grounded diagnosis I can reinforce, **so that** I recall how similar incidents were fixed before instead of re-debugging from scratch.

_This plan delivers the reinforcement + visibility slices: Accept a fix and watch a re-search prioritize it, and see uploaded releases appear as versioned datasets._

|----------|-------------|
| browser → `/feedback/accept` | Client returns session_id, qa_id, source_dataset from the search response — all could be forged |
| source_dataset → improve() target | A forged dataset name would misdirect graph-weight writes |

| T-02-10 | Info Disclosure | feedback.py / datasets_router.py error paths | low | mitigate | logger.exception server-side; short human message to client (D-24) — never str(exc) |
